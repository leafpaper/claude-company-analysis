# 定性分析框架 (Qualitative Analysis Frameworks)

> 融合张磊《价值》核心思想与 VC 定性方法论，为量化评分提供深层判断叠加。

---

## 一、结构性价值评估 — 张磊《价值》

### 1.1 行业结构性变化判断

评估公司是否站在结构性变革的浪潮上，而非仅仅捕捉周期性波动。

**五问判断法：**
1. 这个行业是否正经历不可逆的技术/监管/人口结构变化？
2. 变化的驱动力是外生的（政策/技术突破）还是内生的（消费习惯演变）？
3. 变化的时间尺度是 3-5 年还是 10-20 年？
4. 这个变化是否会重塑产业链的价值分配？
5. 公司在变化中处于受益方还是被颠覆方？

**判定标准：**
- 结构性机会（+2）: 不可逆变革 + 10 年以上时间尺度 + 公司处于价值重塑的核心位置
- 准结构性机会（+1）: 变革方向明确但时间尺度或公司定位存在不确定性
- 周期性/存量博弈（0）: 行业成熟，公司在存量市场中争夺份额
- 逆结构（-1）: 行业面临结构性衰退或技术替代

### 1.2 "大雪长坡"测试

张磊核心投资哲学：寻找"长长的坡，厚厚的雪"——即长期增长空间（坡）和持续积累的竞争优势（雪）。

| 维度 | 长坡（空间） | 短坡（局限） |
|------|-----------|-----------|
| 市场天花板 | TAM 持续扩大，新场景不断涌现 | TAM 固定或萎缩 |
| 增长可持续性 | 渗透率<20%，多个增长引擎 | 渗透率>50%，依赖单一驱动 |
| 价值链延伸 | 可向上下游扩展，平台化潜力 | 单点产品，扩展路径有限 |

| 维度 | 厚雪（积累） | 薄雪（脆弱） |
|------|-----------|-----------|
| 竞争优势 | 随时间自我强化（飞轮效应） | 静态优势，易被追赶 |
| 客户粘性 | 转换成本高，使用越久越依赖 | 低转换成本，随时可替代 |
| 数据/网络 | 数据量越大产品越好（正反馈） | 无数据或网络效应 |

### 1.3 价值创造 vs 价值转移

区分公司是在**创造新价值**（做大蛋糕）还是**转移已有价值**（抢别人的蛋糕）。

**价值创造的信号：**
- 公司产品使此前不可能的事情变为可能
- 用户/客户的总支出在增加（而非从竞品转移）
- 行业整体 TAM 因公司存在而扩大
- 消费者剩余（consumer surplus）在增加

**价值转移的信号：**
- 核心竞争力是"更低价格"而非"更好产品"
- 市场增长主要来自从竞品夺取份额
- 行业陷入价格战或补贴战
- 没有创造新的消费场景

**评分影响：**
- 强价值创造者: +1（维度 1 和维度 2）
- 混合型: 0
- 纯价值转移: -1（维度 1 和维度 2）

## 二、动态护城河分析 — 张磊《价值》

### 2.1 护城河类型与动态性

| 护城河类型 | 静态/动态 | 自然趋势 | 典型案例 |
|-----------|---------|---------|---------|
| 专利/牌照 | 静态 | 到期衰减 | 医药专利、金融牌照 |
| 品牌认知 | 半动态 | 需持续投入维护 | 消费品品牌 |
| 规模经济 | 半动态 | 随规模增强但有上限 | 制造业、物流 |
| 网络效应 | 动态 | 随用户增加自我强化 | 社交平台、支付 |
| 数据飞轮 | 动态 | 数据越多产品越好 | AI/推荐系统 |
| 生态系统 | 动态 | 开发者/合作伙伴锁定 | 操作系统、芯片平台 |
| 转换成本 | 动态 | 客户集成越深越难离开 | 企业SaaS |

### 2.2 护城河趋势评估

关键问题：**护城河是在加深还是在侵蚀？**

**加深信号（+1）：**
- 用户/客户数量持续增长且边际成本下降
- 第三方开发者或合作伙伴在增加
- 客户的使用深度在增加（更多模块/功能/场景）
- 竞品的追赶难度在增加而非减少

**侵蚀信号（-1）：**
- 开源或标准化降低了技术壁垒
- 大厂用更多资源进入赛道
- 客户多供应商策略稀释粘性
- 监管变化降低准入门槛

## 三、"人"的深层评估 — 张磊"人是最重要的"

超越简历和头衔，评估创始人的内在素质。

### 3.1 评估维度

| 维度 | 优秀信号 | 警示信号 |
|------|---------|---------|
| **领导力进化** | 从工程师→管理者→领导者成功转型，能吸引和留住比自己强的人 | 事必躬亲，核心高管频繁流失 |
| **学习速度** | 快速适应新挑战（如从技术转商业），承认错误并迅速纠正 | 重复犯同类错误，拒绝外部建议 |
| **诚信信号** | 公开承诺与实际行动一致，危机时优先保护团队和客户 | 对投资人和员工说法不一，过度美化数据 |
| **第一性原理** | 从问题本质出发而非模仿竞品，有独特的行业洞察 | 跟风型创业，"别人做了所以我也做" |
| **格局与韧性** | 长期主义，拒绝短期利益诱惑，低谷期能坚持 | 急于套现，频繁追逐风口 |

### 3.2 评估方法

- **创始人访谈/公开演讲**：寻找第一性原理思维和深度行业洞察的证据
- **前员工/合作伙伴评价**：脉脉/Glassdoor/社交媒体中的真实反馈
- **行为一致性**：对比不同时期的公开承诺与实际进展
- **危机应对**：研究公司遇到困难时创始人的决策和沟通方式

## 四、VC 定性分析增强框架

### 4.1 创始人-市场匹配度 (Founder-Market Fit)

| 维度 | 评估问题 | 强匹配信号 |
|------|---------|-----------|
| **独特洞察** | 创始人对问题的理解是否超越同行？ | 来自一线经历的深度认知，非二手资料 |
| **不公平优势** | 为什么是这个人来做？ | 独有资源/关系/技术/数据 |
| **执行记录** | 团队能否把想法变成产品变成收入？ | 已有产品发布和客户验证的记录 |
| **时机感知** | 为什么是现在？ | 清晰指出催化剂并被市场验证 |

评分：强匹配（+1）/ 中等匹配（0）/ 弱匹配（-1）

### 4.2 Porter 五力 VC 适配版

传统 Porter 五力 + VC 场景调整（大厂入场和资本涌入权重 2x）：

| 力量 | VC 场景解读 | 权重 |
|------|-----------|------|
| 供应商议价力 | 关键人才、云基础设施、数据供应商的议价能力 | 1x |
| 客户议价力 | 客户集中度、转换成本、价格敏感度 | 1x |
| **新进入者威胁** | **VC 资金大量涌入赛道、大厂宣布进入** | **2x** |
| 替代品威胁 | 相邻技术替代、监管变化催生替代方案 | 1x |
| 行业竞争烈度 | 已融资竞品数量、差异化程度、市场成熟度 | 1x |

整体评估：竞争环境有利（+1）/ 中性（0）/ 不利（-1）

### 4.3 技术 S 曲线定位

判断公司/市场在技术采纳生命周期的位置：

```
创新者(2.5%) → 早期采纳者(13.5%) → [鸿沟] → 早期多数(34%) → 晚期多数(34%) → 落后者(16%)
```

| 阶段 | 投资含义 | 修正值 |
|------|---------|--------|
| 创新者阶段 | 极高风险，技术可能失败，但回报最大 | -1（除非有独特技术壁垒） |
| 早期采纳者 | 高风险高回报，需验证是否能跨越鸿沟 | 0 |
| **跨越鸿沟中** | **最佳投资窗口：风险在降低，增长在加速** | **+1** |
| 早期多数 | 增长确定性高但竞争加剧，需看护城河 | 0 |
| 晚期多数+ | 增长放缓，除非公司有新曲线 | -1 |

### 4.4 网络效应与生态系统分析

| 网络效应类型 | 定义 | 强度评估 |
|------------|------|---------|
| 直接网络效应 | 同类用户越多越有价值（电话、社交） | 用户增长率 + 留存率 |
| 间接网络效应 | 一侧用户吸引另一侧（平台：买家↔卖家） | 双边增长 + 匹配效率 |
| 数据网络效应 | 数据量越大产品越好 | 数据量增长 vs 产品改善的相关性 |
| 生态网络效应 | 第三方开发者/合作伙伴丰富度 | 开发者数量 + API调用量 |

临界点评估：是否已达到自增长飞轮（不依赖补贴也能增长）？
- 已达临界点（+1）/ 接近临界点（0）/ 远未达到（0，不扣分）

### 4.5 监管护城河评估

| 场景 | 影响 | 修正值 |
|------|------|--------|
| 监管壁垒保护在位者，公司已合规 | 新进入者需数年才能获得同等资质 | +1 |
| 监管中性或正在演变 | 不确定性高但非威胁 | 0 |
| 监管可能放松，降低准入门槛 | 公司优势被稀释 | -1 |
| 监管风险：政策可能直接打击业务模式 | 生存威胁 | -1 |

## 五、定性修正系数计算方法

### 5.1 各框架修正值汇总

| # | 框架 | 修正范围 | 适用维度 |
|---|------|---------|---------|
| 1 | 结构性价值（行业变革 + 大雪长坡 + 价值创造） | -2 到 +2 | 整体 |
| 2 | 动态护城河（加深/侵蚀） | -1 到 +1 | 整体 |
| 3 | 创始人深层评估 | -1 到 +1 | 整体 |
| 4 | 创始人-市场匹配度 | -1 到 +1 | 整体 |
| 5 | S 曲线定位 | -1 到 +1 | 整体 |
| 6 | 网络效应 | 0 到 +1 | 整体 |
| 7 | 监管护城河 | -1 到 +1 | 整体 |

### 5.2 计算规则

1. 逐一评估每个框架，给出修正值并附证据
2. 计算**所有适用框架修正值的平均值**
3. 上限封顶：修正系数 ∈ [-1.5, +1.5]
4. 最终调整分 = 量化综合分 + 定性修正系数
5. 报告中必须同时展示：量化分、定性修正、调整后分数

### 5.3 投资信号（调整后）

| 调整后综合分 | 信号 |
|------------|------|
| 8.0+ | 强烈看好 |
| 6.5-7.9 | 有条件看好 |
| 5.0-6.4 | 谨慎 |
| <5.0 | 建议放弃 |

### 5.4 使用注意

- 定性修正不是"拍脑袋加分"——每个修正值必须附带具体证据
- 如果某个框架的信息严重不足（如完全不了解创始人），该框架不参与平均值计算
- 定性修正主要用于处理量化评分难以捕捉的"软信号"——如行业拐点、领导力品质、生态位
