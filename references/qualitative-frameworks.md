# 定性判断框架 v3（Qualitative Judgment Frameworks）

> v3 重大变更：**删除打分制**（v1/v2 的 "-2 到 +2 / -1 到 +1" 系数相加制）。
>
> 定性判断不再给分数，而是输出**逻辑三段式**：核心问题 → 判断 → 证据链 → 逻辑蕴含。
>
> 理由：v1/v2 的打分制让定性分析沦为"随便评个分然后加总"，伪定量化。v3 要求每个框架给出明确的逻辑判断+证据链+投资含义，**直接参与综合结论的论证**，而非数值叠加。
>
> 10 维度评分（scoring-rubric.md）保持不变——那是"事实性打分"（ROE、PE 等可量化指标）。本文件对应的是"深层判断"（护城河、S 曲线、治理等软信号），这些不能用分数表示。

---

## 统一输出模板（v3.1 — 禁止隐性打分）

每个框架的输出**必须**严格遵守：

```markdown
### 框架 N: {框架名}

**核心问题**: {一句话，判断性问题}

**判断**: [是 / 否 / 部分是 / 数据不足]

**证据链**（至少 3 条，每条带来源）:
  1. {Phase 1 的 Tushare / PDF / WebSearch 数据证据}  [来源: ...]
  2. {交叉验证数据或反例}  [来源: ...]
  3. {存疑点或边界条件}  [来源: ...]

**逻辑蕴含**（对投资结论的影响 — ★仅论方向，不论幅度）:
  若判断成立 → {只写方向性影响：利好估值的延续性 / 利好长期竞争优势 / 利空毛利率可持续性 / ...}
  若判断推翻 → {反向影响}

**对综合投资观点的支撑方向**: [看多 / 看空 / 中性 / 分歧]

**未解决问题**（传给 Phase 5 缺口补查）: {若有——如 "子公司 A 未披露分部毛利，无法验证"}
```

**v3.1 禁止清单（严格执行）**:
- ❌ 写"修正值 +1"、"-0.5"等数字
- ❌ 写"调整后综合分 = 量化分 + 修正系数"、"平均修正值"
- ❌ 在"逻辑蕴含"中写具体百分比修正（如 "PE 溢价 20%"、"CAGR 下调 Y%"、"护城河持续 N 年"）——这些是**打分换壳**
- ❌ 写"强烈看多/有条件看好/观望/谨慎/回避"这 5 档排序——这是 5 档隐性打分
- ❌ 用"5 个框架 vs 2 个框架"做票数计数得出综合结论
- ❌ 使用"中性"以外的任何中间态（只能是看多/看空/中性/分歧）

**允许的逻辑蕴含表述**（只论方向）:
- ✅ "利好长期估值的稳定性"（方向：+）
- ✅ "利空增长假设的激进空间"（方向：-）
- ✅ "使估值假设的不确定性上升"（方向：不确定性+）
- ❌ "使 PE 上调 20%"（含幅度 → 打分）
- ❌ "护城河持续 5 年"（含具体年限 → 打分）

### 框架 1: 护城河判断（合并原 1+2+4+5+6）

**核心问题**: 公司的可持续竞争优势是在**加深**、**侵蚀**还是**部分是**？

**判断工具**（综合视角）:

- **行业结构性**: 行业是否有不可逆的结构性变革？公司处于受益方还是被颠覆方？
- **护城河类型**: 是静态型（专利/牌照→到期衰减）还是动态型（网络效应/数据飞轮→自我强化）？
- **三时间切片**:
  - 过去 5 年：哪些护城河源起作用？（毛利率/ROE 稳定性证明）
  - 当下：加深（规模效应扩大、用户粘性提升）还是侵蚀（AI 平民化、新进入者）？
  - 未来 5 年：最大威胁是什么？
- **S 曲线定位**: 早期（PS 估值+高 g）/ 中期（PE 20-35x）/ 晚期（股息折现 10-15x）/ 衰退（清算价值）？
- **网络效应**（若适用）: 有无、类型（双边/数据飞轮/社交）、饱和度
- **Porter 五力**: 进入壁垒 + 竞争烈度综合强弱（五力中 ≥3 强 → 行业承压）

**护城河类型参考表**:
| 类型 | 动态性 | 自然趋势 |
|------|:-----:|---------|
| 专利/牌照 | 静态 | 到期衰减 |
| 品牌认知 | 半动态 | 需持续投入 |
| 规模经济 | 半动态 | 随规模增强但有上限 |
| 网络效应 | 动态 | 自我强化 |
| 数据飞轮 | 动态 | 数据多→产品好 |
| 生态系统 | 动态 | 合作伙伴锁定 |
| 转换成本 | 动态 | 客户集成越深越难离开 |

**证据来源**: Phase 1 §3 产品技术 + §6 竞争格局 + §10 行业宏观 + metrics.json ROE 趋势 + 可比公司毛利率分布 + PDF MD&A 行业段。

**典型逻辑蕴含**（只论方向，不论幅度）:
- 判断"加深（含结构性受益+动态护城河）" → 利好长期超额 ROE 假设 / 利好终值倍数 / 利好抗周期能力
- 判断"侵蚀（行业结构性衰退或护城河被挑战）" → 利空终值倍数 / 利空长期 ROE / 要求估值折价
- 判断"部分是（多业务/多 S 曲线混合，如玩具成熟+游戏早期）" → 强制 SOTP 分部估值

### 框架 2: 管理层判断（合并原 3+4）

**核心问题**: 管理层的 **德**（本分/诚信/股东为伴）、**才**（战略/执行/资本配置）、**创始人-市场匹配度**（若创业公司）是否都合格？

**判断工具**:

- **德** 检验:
  - 是否有年报差错、审计意见瑕疵、诉讼记录？
  - 减持/质押时序是否与股权激励/利空消息时间戳巧合？
  - 承诺履约率：公开讲过的目标是否达到？
  - 关联方交易是否透明？

- **才** 检验:
  - 资本配置历史：并购是否创造价值（看 ROIC 变化）？分红/回购 ROE 是否高于再投资 ROE？
  - 战略定力：5 年前说的战略方向今天还在做吗？
  - 组织能力：核心团队稳定性、创始人是否过度集权？
  - 资本结构决策：是否合理使用杠杆 / 股权稀释节奏是否克制？

- **创业公司专项**（若适用）:
  - 创始人-市场匹配度：创始人背景是否对口该市场？
  - 团队多元性：技术+商业+行业三极是否完整？

**证据来源**: Phase 1 §5 团队 + §7 风险与负面信号 + §9 条款/交易 + `stk_managers` / `stk_rewards` / `pledge_detail`（v4 新用）+ PDF 风险章节。

**典型逻辑蕴含**:
- "德"不过关（减持+年报差错+折价激励 3 信号叠加） → **触发致命看空**（即便基本面好也不投）
- "德"过关+"才"过关 → 利好 PE/PB 溢价空间
- "德"过关+"才"一般 → 利好维持行业均值
- "才"过关+"德"有瑕疵 → 警告"长期警惕背后捅刀"，估值需要治理折价

### 框架 3: 催化剂判断（v4 新增，前瞻性视角）

**核心问题**: 下一个 **12 个月** 是否有可能**改变投资结论方向**的具体、可观察事件？

**判断工具**:

- **内部催化剂**（公司主动）:
  - 新产品/新业务拐点（如实丰文化 AI 玩具爆款验证）
  - 并购/分拆/重组（如子公司独立上市）
  - 股权激励解锁 / 回购启动
  - 重大产能投产 / 订单签订

- **外部催化剂**（环境驱动）:
  - 政策/监管变化（如集采、数据合规）
  - 行业技术拐点（如 AI 降本、新能源技术突破）
  - 利率/汇率宏观周期
  - 重大行业事件（如头部竞品暴雷/被收购）

- **时间窗催化剂**（定时）:
  - 下次财报披露日（`disclosure_date` API，v4 新用）
  - 业绩预告披露窗口（`forecast_vip`）
  - 限售解禁/锁定期到期
  - 股东大会重要议案表决

- **证伪催化剂**（若发生则洞察失败）:
  - 与 Phase 5 差异化洞察的"证伪条件"一致
  - 如"若 2025 年报资产减值 < 2000 万则假设失败"

**证据来源**: Phase 1 的 `disclosure_date` / `forecast_vip` / `express_vip`（v4 新用）+ WebSearch 近期公告 + 行业新闻 + PDF 管理层指引 + Phase 5 证伪条件汇总。

**典型逻辑蕴含**:
- 近期（≤ 3 个月）有 ≥2 个正向催化剂 + 基本面支持 → 利好**短期上行窗口**
- 近期有 ≥1 个已知证伪催化剂风险 → 警告**事件驱动式下行风险**
- 无明显催化剂 + 估值合理 → 中性方向，需耐心等待
- 催化剂密集期（如年报+业绩预告+股东大会连续触发）→ 波动性放大，Phase 7 监控可在此期间密集触发

**v4 催化剂 → Phase 7 监控接口**:
本框架识别出的催化剂列表，会作为 Phase 7 `phase7-quantitative-monitor.md` 的 **"下次监控触发点"** 预设清单。

## 综合判断输出规范（v3.1 — 黑白分割，禁止连续评级）

所有框架评估完毕后，必须在 `phase3-analysis-report.md` 的 §十三 "定性判断"章节输出一段**综合判断陈述**。

### ★黑白分割规则（替代 5 档连续评级，v4 更新框架编号）

为彻底杜绝"5 档变相打分"，综合结论只能是以下 **3 档离散值之一**：

| 综合结论 | 判断标准（严格门槛） |
|---------|-----------|
| **看多方向** | 3 个框架中 **≥ 2 个判定为"看多方向"** **且无"致命看空"** |
| **看空方向** | 3 个框架中 **≥ 2 个判定为"看空方向"** **或** 任一"致命看空"（见下） |
| **中性/分歧** | 其余所有情况（1:1:1 平局、或多个框架数据不足） |

**致命看空条款**（v4.1 简化：**直接引用 Phase 3 §三快筛结果**，不再重复定义）:

§三 快筛 6 项（v4 起）任一触发 → 综合 = 看空，不论其他框架怎么说：
1. 非标审计意见
2. 已证实的财务造假/会计欺诈
3. 商业模式不可持续
4. 持续经营重大疑虑
5. 重大法律/合规可能停业
6. **Audit ≥2 个 🔴 致命级红旗**（v4 新增，`audit_report.md` 自动识别）

### ★禁止项

- ❌ 使用"强烈看多""有条件看好""谨慎""建议回避"等 5 档连续排序
- ❌ 在综合结论里表述"综合分 X.Y"、"定性修正 ±Z" 等数字
- ❌ 用票数加权（如 "4:2 看多胜出" 已经是黑白分割的公开规则——允许；但不能再在此基础上"加权看多程度"做打分）

## 与量化评分的关系

| 维度 | 量化评分（scoring-rubric.md） | 定性判断（本文件） |
|------|-----------|-----------|
| 本质 | 事实性打分（可查可算） | 逻辑判断（需思考） |
| 输出 | 1-10 分 | Yes/No/Partial + 证据链 + 蕴含 |
| 是否加权相加 | ✅ 是（10 维度加权综合分） | ❌ **否**（v3 删除打分制） |
| 用途 | 得出综合分 4.10/10 | 论证投资结论的**方向性**（看多/看空/观望）和**修正空间** |

**重要原则**: 定性判断**不修改综合分的数字**。它修改的是**投资结论的方向**——例如综合分 5.5，但定性判断里 5 个框架都指向看空（治理瑕疵 + 护城河侵蚀 + Porter 五力恶化），那结论就是"观望 / 建议回避"而非"有条件看好"。

这避免了 v1/v2 的"伪定量化"问题——定性修正 -0.5 这种数字看起来精确，实际是主观判断。
